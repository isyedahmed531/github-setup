1. Install Git on Windows + Create a GitHub Account

  1. Go to https://git-scm.com/download/win
    The download starts automatically.
  2. Run the downloaded file → click “Next” all the way.
    Important: When you see “Choosing the default editor”, you can leave it or choose “Visual Studio Code” if you want.
  3. Finish the installation.
  4. Open Git Bash (search in Start menu) or just open VS Code → Terminal → New Terminal (it will use Git automatically).
  5. Create a free GitHub account (if you don’t have one):
    github.com → Sign up

2. Tell Git Who You Are (do this once)

Before you can push code, Git needs your name and email (the same one you used on GitHub).

Open VS Code Terminal (or Git Bash) and paste these two lines one by one: 

git config --global user.name "Your Full Name" git config --global user.email "your-email@example.com"

Replace the text inside quotes with your real name and the email you used for GitHub.

3. Clone a Repository (bring the project to your computer)

Way 1 – Using VS Code File Explorer (easiest)

  1. Open VS Code → press Ctrl + Shift + P (Command Palette)
  2. Type “Git: Clone” and choose it
  3. Paste the repository URL (example: https://github.com/username/repo-name.git)
  4. Choose the folder on your computer where you want it
  5. Click “Open” when it finishes

Way 2 – Using the Terminal

# Go to the folder where you want the project cd Desktop          # example: go to Desktop git clone https://github.com/username/repo-name.git code repo-name      # this opens the folder in VS Code

4. Push and Pull Changes

Using VS Code Buttons (recommended for beginners)

  • Open the folder in VS Code
  • Click the Source Control icon on the left (looks like a branch ↔)
  • You’ll see changed files → click “+” to stage them
  • Type a message at the top (e.g. “Add login page”) → press Ctrl + Enter to commit
  • Click the ⋯ menu → Pull (to download latest) or Push (to upload)

Using Terminal (copy-paste)

# Save your changes git add . git commit -m "Your message here" # Upload to GitHub git push # Download latest changes from GitHub git pull

The first time you push with HTTPS, Git will ask for username and a Personal Access Token (not your normal password). See the next section if you get stuck.

5. Work from Multiple Computers (Home PC + Laptop + Office)

GitHub is the “cloud copy”. Just remember the golden rule:

  1. Always PULL first when you sit down → gets the newest files
  2. Make your changes
  3. Commit + Push when you finish

Do the same on every computer and you will never lose work or get big conflicts.

If VS Code says “conflict”, it will highlight the files — just choose which version to keep (yours or the one from the cloud).

6. “This repository is from someone else, please fork” or login keeps failing

If GitHub says you need to fork

  1. Go to the repository page on github.com
  2. Click the Fork button (top-right)
  3. Now clone your fork instead of the original
  4. Work normally → push to your fork → open a Pull Request when ready

Fix wrong password / “fatal: Authentication failed”

GitHub no longer accepts your normal password. You need a Personal Access Token (PAT).

  1. Go to github.com/settings/tokens → “Fine-grained tokens” or “Tokens (classic)” → Generate new token (classic)
  2. Give it a name, select “repo” scope, click Generate
  3. Copy the token (it looks like ghp_xxxxxxxxxxxxxxxx)

Next time you push:

  • Username → your GitHub username
  • Password → paste the token (nothing appears when you paste — that’s normal)

Still not working? Clear old saved password

  1. Press Win + S → type “Credential Manager” → open it
  2. Click “Windows Credentials”
  3. Look for entries that start with git:https://github.com
  4. Click → Remove
  5. Try push again → it will ask for the token again
